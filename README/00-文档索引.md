# 文档索引

## 重要度说明

- `S级`：决定项目方向、产品边界和架构原则的核心文档，必须优先遵循
- `A级`：执行层文档，用于排期、设计、拆解、追踪、验收和质量收口
- `B级`：历史参考文档，保留最初需求背景，不直接代表当前最优路线

## 推荐阅读顺序

1. 先看 `S1-产品大纲`
明确项目最终要做成什么，以及当前最大的产品和架构风险。

2. 再看 `A2-数据架构重构规划`
明确当前代码是否足以支撑后续玩法扩展，以及数据应该如何映射到 UI。

3. 然后看 `A5-游戏框架设计`
明确框架如何承接职业体验、战斗事件和玩家可读的展示层。

4. 再看 `A1-后续开发计划`
明确接下来应按什么顺序推进版本。

5. 然后看 `A3-开发规则` 和 `A4-进度追踪`
确保执行方式、质量门槛和记录方式统一。

6. 最后看 `A6-职业设计方案`
将职业体验与数据、框架、UI 承接关系对照起来理解。

7. `B1-初版计划书`
仅作为历史参考，不作为当前开发主依据。
