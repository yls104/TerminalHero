# 文档索引

## 重要度说明

- `S级`：决定项目方向、产品边界和架构原则的核心文档，必须优先遵循。
- `A级`：执行层文档，用于排期、设计、拆解、追踪和验收。
- `B级`：历史参考文档，保留最初需求背景，不直接代表当前最优路线。

定义项目目标、完成标准、当前不足、最终产品形态与必须前置的架构原则。

2. `A级` [A1-后续开发计划.md](/F:/HTML2DRPG/README/A1-后续开发计划.md)

6. `B级` [B1-初版计划书.md](/F:/HTML2DRPG/README/B1-初版计划书.md)

## 推荐阅读顺序

1. 先看 `S1-产品大纲`，明确项目最终要做成什么，以及现阶段最大的风险在哪里。
2. 再看 `A2-数据架构重构规划`，明确当前代码是否能支撑后续玩法扩展。
3. 然后看 `A1-后续开发计划`，明确接下来应该按什么顺序推进。
4. 再看 `A3-开发规则` 和 `A4-进度追踪`，确保执行方式和记录方式统一。
5. 最后看 `B1-初版计划书`，对照项目从最初 MVP 到当前路线的演进差异。
