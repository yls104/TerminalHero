# S级文档：产品大纲

## 1. 项目定位

- 项目名称：`Terminal Brave`
- 当前形态：原生前端实现的 2D 俯视角探索 + 回合制 RPG 原型
- 目标形态：具备稳定单轮冒险闭环、可扩展内容生产能力、可在手机横屏稳定游玩的轻量 RPG

这个项目接下来不能再只按“把功能做出来”推进，而要按“能不能形成持续游玩价值”推进。

## 2. 当前已有基础

- 城镇、职业、技能、装备、战斗、关卡、Boss、回城循环已经具备雏形。
- 手机横屏版本已经有独立布局和触控方案，不再是桌面界面硬缩。
- 关卡已经从固定地图升级为程序化生成，并且开始有主题差异。
- 不同区域的小怪和 Boss 已经有基础数值区分与战斗角色区分。

这些说明项目已经从“单一玩法 demo”走到了“系统原型”阶段。

## 3. 当前最核心的不足

### 3.1 核心循环还不够强

现在玩家能做的是：

- 进图
- 清怪
- 打 Boss
- 回城

但这条链路还缺一个关键问题的答案：玩家为什么愿意再来一轮？

目前不足在于：

- 掉落期待感不强
- 回城成长的反馈还偏弱
- 刷旧图和推新图的收益差异不清晰
- 单轮冒险中的事件变化还不够多

如果不补这一层，程序化关卡只会变成“随机重复”，不会变成“可重复游玩”。

### 3.2 养成系统还停留在浅层

当前有职业、技能、装备、技能点，但离真正的 RPG 构筑还差一层：

- 技能树或职业专精
- 装备词条或装备成长
- 明确的流派分化
- 长线资源目标

现在更像“玩家每轮都变强一点”，而不是“玩家正在打造自己的 build”。

### 3.3 程序化生成还主要停留在地图层

目前随机的是：

- 地形
- 怪物点位
- 区域主题

真正成熟的程序化内容还应该继续发展到：

- 房间功能差异
- 事件节点
- 精英怪与奖励点
- 可选支线
- 风险收益选择

否则后面即使加更多素材，玩家仍然会感受到结构单一。

### 3.4 城镇系统还不像真正的 Hub

现在城镇承担了职业、商店、出征入口这些功能，但还不像一个有生命力的中心区域。

后续城镇应该承担：

- 成长反馈
- 资源消耗
- 长期目标
- 世界推进感

如果城镇只是菜单集合，那么“回城”就只是流程切换，不是情绪节点。

### 3.5 工程结构还不够支撑长期扩展

现在项目能继续加功能，但技术债已经开始露头。

关键风险：

- `main.js` 体量偏大，承担了过多系统职责
- 游戏状态、UI 状态、关卡状态没有完全分层
- 数据配置还不够彻底外置
- 缺少存档体系
- 缺少可重复执行的验证链路

如果继续在现状上直接堆玩法，开发速度会越来越慢，回归成本会越来越高。

## 4. 距离真正完成还差什么

我把“真正完成”定义成以下 3 个标准：

1. 玩家有连续玩 3 到 5 轮的理由。
2. 开发侧新增一个职业、地图、怪物、事件时成本可控。
3. 手机横屏实机体验稳定，不需要人为兜底修正。

按这个标准判断，项目现在更接近“`系统原型 40%`”，而不是“`可交付成品 80%`”。

问题不在于缺几个功能，而在于系统之间还没有形成真正的乘法关系。

## 5. 接下来的产品原则

### 原则一：先做强单轮冒险，再扩内容池

先把一轮体验做得扎实：

- 进图
- 遭遇普通战
- 遭遇精英/事件
- 获得奖励
- 打 Boss
- 回城构筑
- 选择下一轮

这条链路扎实了，再扩职业、地图、事件才有意义。

### 原则二：每个新系统都必须增强“选择”

后续新增内容不能只是“数值更高”或“按钮更多”。

必须至少增强其中一项：

- 战斗中的选择
- 路线中的选择
- 成长中的选择
- 资源分配中的选择

### 原则三：优先建设内容生产能力

项目以后最值钱的不是“再写死一张图”，而是建立一套能持续产出内容的结构。

优先建设：

- 数据配置表
- 关卡规则模板
- 事件模板
- 敌人模板
- 奖励模板

这样后面扩内容才不会每次都动主逻辑。

### 原则四：移动端体验继续保持主视口优先

横屏移动端已经成为项目的重要目标平台，后续所有新 UI 都必须默认先考虑：

- 主视口占比
- 左手触达
- 操作层级
- 文案可读性
- 单轮操作负担

## 6. 我认为最值钱的下一步

不是立刻继续加第 8 个职业，也不是继续做一套新皮肤。

最值钱的一步是把“单轮冒险闭环”做成真正有驱动力的版本：

- 普通战
- 精英战
- 事件节点
- 奖励选择
- Boss 前准备
- 回城结算
- 下一轮构筑分支

只要这一步做实，项目就会从“系统集合”跨进“游戏”。

## 7. 当前版本后的目标状态

下一阶段版本应该努力达到：

- 玩家可以明确区分不同区域的玩法差异
- 玩家一轮结束后会期待回城强化和下一轮出征
- 玩家会开始讨论职业流派，而不是只讨论数值高低
- 开发者能更低成本地继续扩关卡、敌人、事件和装备

达到这个状态后，再去谈更大规模的内容拓展，才是真正站得住的路线。
