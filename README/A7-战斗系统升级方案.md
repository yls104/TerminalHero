# A级文档：战斗系统升级方案

## 1. 目标

当前战斗系统已经能支撑：

- 普通战 / 精英战 / Boss 战
- 职业资源池
- 专精强化
- 遗物联动
- 状态效果与技能说明

但它仍然本质上是“轮流行动”的回合制。  
这会带来两个明显问题：

- `速度` 的存在感偏弱，主要只影响先手和撤退
- “多动症”“抢节奏”“插队爆发”这类玩法很难成立

因此，正式版前需要对战斗系统进行一次方向正确的大改：  
把当前的轮流回合制，升级为轻量化的 `行动值（AV）时间轴系统`，并接入 `终结技插入系统`。

## 2. 这次升级不追求什么

这轮升级不是为了复制某一款商业作品的全部复杂度。  
它的目标不是：

- 完整复刻《轨迹》的多单位复杂时间轴
- 完整复刻《崩坏：星穹铁道》的全套击破与角色协同
- 完整复刻《光与影：33号远征队》的完整反应式演出与系统深度

这轮真正要做的是：

- 让速度成为战斗中的强属性
- 让行动顺序本身变成决策对象
- 让终结技具备明确的“插入时机”与操作感
- 让战斗从“轮流放技能”升级为“围绕节奏与窗口抢轴”

## 3. 新战斗系统的核心体验

## 3.1 行动值时间轴

每个战斗单位不再简单依赖 `playerTurn = true/false`。  
而是拥有自己的 `行动值` 或 `到下次行动的剩余时间`。

基本规则建议如下：

- 每个单位都有 `currentAv`
- `currentAv` 归零或达到行动阈值时，获得行动权
- 技能具有 `基础延迟`
- 速度会影响延迟结算或行动值推进效率
- 加速 / 减速 / 提前行动 / 延后行动都直接作用于时间轴

这样 `速度` 才不是边角修饰，而是节奏核心。

## 3.2 终结技插入

终结技不再完全等同于普通技能。  
建议定义为：

- 拥有独立的终结技资源
- 满足条件后可在敌方行动前、己方行动后或关键窗口中插入
- 终结技插入后不直接吞掉普通行动，但会带来明确的资源或时间轴代价

目标体验是：

- 玩家会主动等待更好的插入时机
- 玩家会为了“插入后改变行动序列”而保留终结技
- 终结技不只是大数字，而是“改写节奏”

## 3.3 技能不再只看伤害，还要看节奏代价

新系统下，技能至少有三类收益：

- 数值收益
- 资源收益
- 节奏收益

例如：

- 高伤技能：伤害高，但延迟更长
- 连段技能：伤害一般，但延迟短、能更快回轴
- 控场技能：伤害不高，但能让目标行动延后
- 启动技能：当下收益一般，但能为下一次终结技或爆发窗口铺路

## 4. 这套系统会带来的产品价值

## 4.1 速度属性终于“活起来”

当前速度更多是隐藏在底层公式里的弱修饰。  
新系统下，速度会直接影响：

- 谁更快上轴
- 谁能更快打第二次
- 谁更容易抢在 Boss 前插技能
- 谁更适合短延迟循环

## 4.2 “多动症”构筑终于有土壤

现在想做“高频行动”“抢节奏”“先手连击”类玩法，会很别扭。  
AV 系统接入后，这类体验会自然成立：

- 短延迟 build
- 连续行动 build
- 终结技插入 build
- 延后敌人行动 build
- 高速资源循环 build

## 4.3 战斗深度提升，但直觉也更强

如果只看数值倍率，战斗深度容易停留在“这招强不强”。  
引入时间轴后，玩家会开始思考：

- 我现在出这招值不值
- 我会不会因此丢掉下一个窗口
- 我能不能把敌人的关键回合往后推
- 我该不该把终结技留给下一次插入

这比“轮到我就按最强技能”更有策略感。

## 5. 范围控制原则

这次升级必须收住范围。  
建议正式版前只做以下几项：

- 单位行动值时间轴
- 技能基础延迟
- 速度影响行动序列
- 加速 / 减速 / 提前 / 延后
- 终结技独立资源与插入
- 时间轴可视化
- 技能说明里展示节奏信息

这次不要做：

- 多单位站位系统
- 复杂群体协同插队
- 完整弱点击破体系
- 过深的 QTE 反应判定
- 职业数值大规模平衡

## 6. 新系统下必须补的 UI

如果没有 UI，这次升级的价值玩家感知不到。  
最低必须补齐：

- `时间轴预览条`
  显示接下来若干个行动者
- `当前单位行动提示`
  明确告诉玩家现在是谁在行动
- `技能节奏信息`
  显示该技能的延迟、是否提前自身、是否延后敌人
- `终结技可插入提示`
  明确告诉玩家“现在可以插入”
- `终结技资源显示`
  不能只在详情页能看，要在战斗上下文中可见

## 7. 这次升级不考虑什么

你已经明确过：  
当前阶段不把职业平衡和最终数值作为主任务。

因此这轮设计可以先忽略：

- 七职业之间的最终强度差
- 各区域敌人的最终数值曲线
- 终结技资源最终获取速度
- 极限 build 的平衡边界

现阶段优先级是：

1. 机制先成立
2. 节奏先好玩
3. UI 先可读
4. 再谈职业和数值平衡

## 8. 新战斗系统的验收标准

这次升级完成后，至少应达到：

- `速度` 成为玩家可以明显感知的有效属性
- 玩家可以通过时间轴理解“为什么现在轮到谁”
- 玩家能通过技能说明理解“这一招会不会拖慢自己”
- 终结技插入成为真实可用的决策，而不是单纯大招按钮
- 玩家会开始思考行动顺序，而不只是思考哪招倍率高
- 不同 build 可以围绕“快轴 / 慢轴 / 插队 / 延后”形成区别

如果做不到这些，那就说明只是换了底层术语，没有真正升级战斗体验。
