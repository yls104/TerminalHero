# A级文档：职业重构计划

## 1. 为什么现在必须重构职业

当前战斗系统已经不再是早期的“你一回合、敌人一回合”模型，而是已经具备：

- AV 时间轴
- 终结技插入
- 韧性 / 失衡 / 蓄力 / 打断
- 敌我双方都能围绕失衡窗口做决策
- 遗物开始围绕 `打断 / 压制 / 处决 / 抢轴` 形成 build

在这种前提下，旧职业方案的主要问题已经非常明确：

1. 职业资源和技能语义还是偏“旧回合制”
2. 很多职业描述没有真正绑定到时间轴和压制轴
3. 不同职业之间的差异仍有不少停留在“倍率不同、资源名不同”
4. UI 目前能承接共享战斗语言，但还不足以承接七职业各自的高级机制

所以现在不应该继续把七职业一起往前硬推，而是应该进入“逐职业重构”阶段。

## 2. 当前执行策略

- 已按顺序完成七职业的首轮重构与恢复开放
- 当前所有职业都已具备可读的资源循环、技能职责与 UI 承接
- 后续不再讨论“能否开放”，而是转入职业二轮深化

首轮阶段的意义，是先把每个职业从旧回合制语义中拉出来，统一接到时间轴、压制轴与构筑系统上。

## 3. 战士为什么先重构

战士最适合作为第一职业，原因有三点：

1. 它天然与 `压制 / 失衡 / 处决` 这套共享战斗语言最贴近
2. 它更容易验证“职业资源 -> 压制窗口 -> 爆发处决”的完整闭环
3. 它能先帮我们验证职业重构所需的数据结构、UI 承接和数值模板

换句话说，战士是最适合做“职业系统新版模板”的职业。

## 4. 战士重构目标

战士重构后，不应该只是“高血高攻职业”，而应该是：

`围绕压制、破势、重击窗口推进的正面压迫型职业`

### 4.1 核心体验

- 用主动压制击穿敌方韧性
- 在蓄力 / 失衡窗口中打出高价值重击
- 用有限但明确的回稳手段撑到下一个爆发窗口

### 4.2 战斗语言映射

- 资源主轴：`压制值`
- 共享战斗轴：
  - `打断`
  - `压制推进`
  - `失衡处决`
  - `爆发窗口`

### 4.3 需要重构的内容

1. 资源循环
   - 旧版压制值更像“普通职业资源”
   - 新版应更明确地与破势、重击、处决窗口挂钩

2. 技能分工
   - 至少拆清 `起手压制`、`破势重击`、`回稳过渡`、`终结处决`
   - 不允许所有技能都只是“伤害倍率不同”

3. 专精路线
   - 一条偏 `破军 / 失衡处决`
   - 一条偏 `坚城 / 顶压推进`

4. UI 承接
   - HUD 要能看出战士是否正处在强势窗口
   - 技能按钮要让玩家读懂“这招是拿来打断、破势，还是拿来处决”
   - 构筑详情要能说明战士当前 build 偏向哪种战斗语言

## 4.4 当前已落地的首轮实现

- `裂风斩` 已改成战士的主起手压制技，负责推进韧性、拆蓄力并快速积累压制值
- `战吼` 已改成低消耗的窗口启动技，不再是单纯倍率按钮
- `裂地猛击` 已改成明确的压制兑现 / 处决收尾技能
- `不屈战吼` 已改成“回稳并重建压制”的坚城路线技能，而不是只回一点血
- 构筑详情与战斗按钮现在会直接提示 `起手 / 起窗 / 处决 / 回稳` 等职责语义

这意味着战士已经不再只是“压制值职业”，而是开始形成完整的压制窗口闭环。

## 5. 其它职业的重构方向占位

当前已完成首轮落地，并保留方向定义，避免后续二轮深化再漂移：

- 法师：围绕 `慢轴高收益 / 反蓄力 / 终结插入`
- 游侠：围绕 `抢轴 / 拖轴 / 持续压制`
- 牧师：围绕 `恢复转收益 / 稳态推进 / 审判收尾`
- 盗贼：围绕 `连续行动 / 斩杀阈值 / 多动爆发`
- 圣骑士：围绕 `稳态防反 / 誓能积累 / 失衡处决`
- 德鲁伊：围绕 `状态轮转 / 资源转化 / 节奏拉扯`

这些方向现在都已进入“已开放试行、后续继续深化”的阶段。

## 6. 职业重构顺序建议

建议按以下顺序推进：

1. 战士
2. 盗贼
3. 游侠
4. 圣骑士
5. 法师
6. 德鲁伊
7. 牧师

原因：

- 先把最依赖时间轴 / 压制轴的职业做完
- 再处理更依赖状态循环与稳态资源的职业
- 最后再收口恢复转收益和更复杂的容错职业

## 7. 开放规则

后续每个职业重新开放选择前，必须同时满足：

1. 已完成该职业的资源循环重构
2. 已完成该职业的技能职责重构
3. 已完成该职业的 UI 承接说明
4. 已完成至少一轮专项验证
5. 已在进度文档中从 `重构中` 改为 `可选`

没有同时满足这五项，不允许恢复选择入口。

当前状态：上述五项已由七职业首轮重构全部满足。后续重点转入职业二轮深化，而不再是是否开放入口。
