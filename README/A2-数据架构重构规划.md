# A级文档：数据架构重构规划

## 1. 目标

当前项目的数据结构只能支撑“原型期玩法”，不足以安全支撑后续的：

- 技能树
- 遗物系统
- 复杂掉落
- 事件节点
- 多阶段关卡
- 存档系统

本规划的目标是提前定义项目最终需要的数据结构边界，避免后续所有开发建立在脆弱底座上。

## 2. 当前结构的不足

### 2.1 玩家数据过于扁平

当前玩家数据大多是单层字段，适合基础战斗，不适合复杂成长。

不足表现：

- 缺少职业成长分支信息
- 缺少临时状态和永久状态的清晰边界
- 缺少遗物、装备词条、资源包等分层结构
- 不利于存档和版本迁移

### 2.2 技能定义还不够抽象

当前技能结构适合少量固定技能，但对以下场景支撑不足：

- 技能树解锁关系
- 技能升级
- 多目标/条件触发
- 状态类技能的统一结算

### 2.3 敌人与关卡配置不够模板化

当前区域和敌人虽然开始有主题，但仍偏“逻辑内嵌 + 局部配置”。

不足表现：

- 区域规则和地图生成规则耦合
- 敌人的掉落、事件权重、生成权重没有统一结构
- 精英怪和 Boss 的差异还没被单独建模

### 2.4 缺少真正的运行态与配置态分离

现在很多数据既承担“静态配置”的角色，又承担“运行时状态”的角色。

这会导致：

- 配置难复用
- 状态难存档
- 测试难隔离
- 功能扩展时容易互相污染

## 3. 最终应具备的数据层次

## 3.1 配置层

只描述“是什么”，不描述“当前变成了什么”。

应包含：

- 职业配置
- 技能配置
- 装备配置
- 遗物配置
- 敌人模板配置
- 区域模板配置
- 事件模板配置
- 掉落模板配置

## 3.2 运行态层

只描述“本轮游戏当前是什么状态”。

应包含：

- 玩家当前属性
- 当前关卡状态
- 当前事件状态
- 当前战斗状态
- 当前临时增益/减益
- 当前资源与背包

## 3.3 持久化层

只描述“哪些内容需要跨轮或跨会话保留”。

应包含：

- 解锁职业/区域
- 玩家长期资源
- 已拥有装备/收藏
- 当前存档位信息
- 版本号与迁移字段

## 4. 建议的核心数据模型

## 4.1 PlayerState

应拆成：

- `identity`
- `baseStats`
- `derivedStats`
- `resources`
- `build`
- `inventory`
- `runtimeStatus`
- `progress`

这样才能清晰区分基础成长、装备修正、临时状态和长期进度。

## 4.2 SkillConfig

应包含：

- 基础信息
- 消耗
- 目标类型
- 结算类型
- 状态效果
- 解锁条件
- 升级关系
- 标签

标签示例：

- `physical`
- `magic`
- `heal`
- `buff`
- `poison`
- `starter`
- `ultimate`

## 4.3 EnemyTemplate

应包含：

- 基础数值
- 行为角色
- 技能池
- 掉落池
- 生成权重
- 所属区域
- 是否精英 / 是否 Boss

## 4.4 StageTemplate

应包含：

- 主题信息
- 美术资源键
- 地图生成规则
- 怪物池
- 精英池
- 事件池
- 奖励池
- Boss 房规则

## 4.5 RunState

这是单轮冒险的核心运行态，建议单独存在。

应包含：

- 当前区域
- 当前层级或章节
- 当前地图实例
- 本轮获得的遗物
- 当前路线事件结果
- 当前奖励记录

## 5. 模块边界建议

## 5.1 Config 模块

负责所有静态配置读取与组合。

## 5.2 Domain 模块

负责纯规则与结算逻辑。

建议至少拆出：

- `combat`
- `run`
- `progression`
- `stage_generation`

## 5.3 UI 模块

只处理显示，不直接承载战斗公式和成长逻辑。

## 5.4 Persistence 模块

负责存档、读档、版本迁移。

## 6. 近期重构优先级

### 第一优先级

- 将职业、技能、敌人、区域定义从流程逻辑中抽离成配置对象
- 区分静态配置和运行时状态

### 第二优先级

- 抽离战斗结算核心
- 抽离关卡生成核心

### 第三优先级

- 为存档留出稳定字段
- 为遗物、事件、掉落预留扩展位

## 7. 验收标准

完成本轮数据结构重构后，应达到：

- 新增一个敌人模板主要通过配置完成
- 新增一个区域模板不需要频繁修改主流程代码
- 战斗逻辑可不依赖 UI 单独描述输入输出
- 玩家状态可明确区分长期进度、单轮进度和临时状态

如果达不到这几点，说明重构还没有真正完成。
