# A级文档：无尽回廊计划

## 1. 立项目标

当前项目已经具备职业、构筑、章节推进和长期沉淀，但当玩家把角色 build 到一个相对成型的阶段后，还缺少一个能持续测试上限、反复刷新纪录的终局玩法。

`无尽回廊` 的目标不是再做一张普通关卡，而是补上“构筑完成后，玩家还能为什么继续玩”的长期驱动力。

它要同时满足：

- 让玩家持续挑战更高层数与更高积分
- 让现有战斗、奖励、存档和城镇循环继续发挥作用
- 让长期纪录和回城沉淀有明确反馈
- 不引入一套与主游戏割裂的独立小游戏框架

## 2. 核心体验

玩家在完成当前三章试炼后，可在城镇守门人处进入 `无尽回廊`。

基础循环如下：

1. 从城镇进入无尽回廊第 1 层
2. 每层只保留一次关键战斗和一个补给点，降低杂讯，突出纯构筑对战斗强度的考验
3. 每层胜利后先领取战后奖励，再决定 `继续下层` 或 `见好就收`
4. 每 5 层进入首领考核层，用更高强度的主题 Boss 检查当前 build 的完整度
5. 玩家可在进入回廊前设置起始层数，强化 build 后可直接从已到达高层继续挑战
6. 主动结算、撤退或战败后，把本轮成绩换算成长期沉淀并回到城镇
7. 在城镇查看最高层、最高分、总挑战次数，再决定是否继续冲榜

## 3. 规则设计

### 3.1 楼层结构

- 普通层：单场常规试炼战，用于稳定累积收益与积分
- 精英层：每 3 层插入一次，压力和分数更高
- 首领层：每 5 层出现一次，检验 build 的抗压、爆发和资源调度能力

### 3.2 区域主题

- 回廊每 5 层切换一次主题区域
- 当前版本直接复用 `青藤密林 / 沉没书库 / 余烬裂谷` 的敌人与美术主题
- 这样既能保证主题差异，也避免为了首版终局模式再额外建设一整套内容池

### 3.3 结算方式

- 每层击破敌人后获得常规战后奖励
- 玩家进入回廊前可自由选择 `1 ~ 历史最高已击破层` 之间的任意起始层
- 跳过的楼层不会补发奖励，最终结算只按本轮实际击破的楼层和积分计算
- 玩家可在每层战后主动结算
- 最终回城时，根据 `层数 + 首领层击破数 + 当前积分` 结算传承印记
- 首领层可额外提供少量城镇声望，保证该模式仍与长期成长闭环相连

## 4. 数据与 UI 承接

### 4.1 持久化数据

新增到 `progress.longTerm.endlessTrial`：

- `bestFloor`
- `bestScore`
- `totalRuns`
- `totalLegacyMarksEarned`
- `totalBossesDefeated`
- `lastFloor`
- `lastScore`
- `lastOutcome`

### 4.2 运行态数据

新增 `currentChallengeRun`，用于记录当前是否正在无尽回廊中、当前楼层、本轮积分、已击破首领层数等。

当前楼层快照中的 `contentPools.challenge` 还应继续补齐：

- `floorType`
  当前层属于普通层、精英层还是首领层
- `affixRule`
  当前层采用的词缀分配规则快照，如抽取数量、候选池和强制词缀
- `affixes`
  当前层实际挂接的词缀快照，至少包含 id、作用对象、触发时机、效果载荷与玩家可读文案
- `affixSummary`
  当前层可直接用于 UI 预警与日志摘要的词缀简报文本

### 4.3 UI 映射

- 守门人入口：展示是否开放、最高层、最高分、总挑战次数
- 守门人入口：进入回廊前可设置起始层数
- 远征摘要：进入回廊后展示当前楼层、本轮积分和历史最好成绩
- 后续高层预警：优先读取 `challenge.affixes` 与 `challenge.affixSummary`，而不是在 UI 层重新拼接规则文本
- 战后浮层：胜利后给出 `继续下层 / 见好就收`
- 构筑详情：在长期沉淀区域展示无尽回廊纪录
- 冒险结算：额外展示本轮层数、积分、首领层击破数和新纪录

## 5. 验收标准

- 玩家能在城镇清楚知道无尽回廊是否开放、最高纪录是多少
- 每层胜利后存在清晰的 `继续 / 结算` 选择，而不是直接强推下一层
- 本轮成绩会稳定写入存档，并在回城后能被再次读取
- 该模式不破坏现有章节流程、战斗流程和存档流程
- 玩家不需要阅读规则文档，也能直觉理解“这是终局冲层模式”

## 6. 首版落地范围

本次首版只做以下内容：

- 城镇入口与开放条件
- 单层关键战斗地图
- 分层敌人与首领层规则
- 本轮积分与长期纪录
- 见好就收结算
- 存档与验证链路接入

暂不在本轮加入：

- 专属排行榜
- 专属遗物池
- 多选房间分支
- 回廊专属事件
- 赛季制轮换词缀

这些内容保留给后续 `V1.3-R2 / R3` 深化阶段继续扩展。
